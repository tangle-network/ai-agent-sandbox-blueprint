# Harden Report — ai-agent-sandbox-blueprint
Date: 2026-04-15

## Fixes Applied (this session)

### 1. CRITICAL: Token revocation bypass via PASETO fallback — FIXED
- **File**: `sandbox-runtime/src/session_auth.rs`
- **Issue**: `revoke_session()` removed a token from the in-memory `SESSIONS` map, but `validate_session_token()` fell back to PASETO cryptographic validation when the token wasn't in the map. A revoked token continued to authenticate for up to 1 hour (SESSION_TTL_SECS).
- **Fix**: Added `REVOKED` blacklist (`Lazy<Mutex<HashMap<String, u64>>>`). `revoke_session()` now adds tokens to the blacklist. `validate_session_token()` checks `REVOKED` before the PASETO fallback. `gc_sessions()` prunes expired blacklist entries. `revoke_sessions_for_address()` also populates the blacklist.
- **Tests added**: 4 new tests (`revoked_token_rejected_via_paseto_fallback`, `revoke_sessions_for_address_blocks_paseto_fallback`, `revocation_blacklist_gc_removes_expired_entries`, `revoke_unknown_token_still_blacklists`)

### 2. HIGH: context_json could override operator-enforced maxTurns — FIXED
- **File**: `sandbox-runtime/src/operator_api.rs:2103-2115`
- **Issue**: `build_agent_payload()` called `metadata.extend(ctx)` AFTER inserting `maxTurns`, allowing user-supplied `context_json` containing `"maxTurns": 999999` to override the operator's turn limit, enabling unbounded agent execution and cost escalation.
- **Fix**: Now extends context FIRST, explicitly strips `maxTurns` from user context, then inserts the operator-enforced value.
- **Tests added**: 2 new tests (`test_build_agent_payload_context_json_cannot_override_max_turns`, `test_build_agent_payload_context_json_without_max_turns_override`)

### 3. HIGH: Instance blueprints bound 0.0.0.0 unconditionally — FIXED
- **Files**: `ai-agent-instance-blueprint-bin/src/main.rs:188`, `ai-agent-tee-instance-blueprint-bin/src/main.rs:194`
- **Issue**: Both instance blueprint binaries unconditionally bound `[0, 0, 0, 0]` (all interfaces), exposing the operator API to any host on the same network. The sandbox blueprint correctly bound `127.0.0.1` behind BPM.
- **Fix**: Now defaults to `127.0.0.1` (loopback only). Set `BIND_ALL_INTERFACES=true` to opt-in to `0.0.0.0` binding.
